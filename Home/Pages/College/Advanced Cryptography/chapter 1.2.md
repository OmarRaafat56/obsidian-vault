## Chapter 8 — Public-Key Cryptosystems Based on the Discrete Logarithm Problem

# Some Algebra

## Groups

* A **group** is a set ( G ) with an operation ( \circ ) satisfying:

  * Closure
  * Associativity
  * Identity element
  * Inverse for each element

* A group is **finite** if it contains a finite number of elements.

**Notation**

* (|G|) → **order (cardinality)** of the group
* Example: ( Z_{11}^* = {1,2,3,...,10} )

---

# Cyclic Groups

A **cyclic group** is a group where all elements can be generated by repeated application of one element (called the **generator** or **primitive element**).


$$G = {1, \alpha, \alpha^2, \alpha^3, ..., \alpha^{n-1}}$$


Where:

* ( \alpha ) = generator
* ( n = |G| )

### Example

Check if **a = 2** is a primitive element of ( Z_{11}^* ).

Compute powers of 2 mod 11:

| Power    | Value |
| -------- | ----- |
| (2^1)    | 2     |
| (2^2)    | 4     |
| (2^3)    | 8     |
| (2^4)    | 5     |
| (2^5)    | 10    |
| (2^6)    | 9     |
| (2^7)    | 7     |
| (2^8)    | 3     |
| (2^9)    | 6     |
| (2^{10}) | 1     |

Since all elements appear → **2 is a primitive element**.

Important property:

**Element orders divide the group cardinality.**

# Subgroups

## Cyclic Subgroup Theorem (Theorem 8.2.5)

If ( (G, \circ) ) is cyclic and
( a \in G ) with order ( s ),

then **a generates a cyclic subgroup with s elements.**

### Example

For ( G = Z_{11}^* )

Possible subgroup sizes:

* 1
* 2
* 5
* 10

These are the **divisors of |G| = 10**.

Example:


$$ord(3) = 5$$


Subgroup generated by 3:


$$H = {1,3,4,5,9}$$


This set is **closed under multiplication mod 11**.

---

# Generating Subgroups (Theorem 8.2.7)

For cyclic group ( G ):

Generator of subgroup:


$$\alpha^{n/k}$$


Where:

* ( n = |G| )
* ( k ) divides ( n )

Number of generators of ( G ):


$$\phi(|G|)$$


Where ( \phi ) = **Euler Totient Function**.

Example:

For ( Z_{11}^* ):

Primitive element = **8**

Use powers of 8 to regenerate subgroups.

# The Discrete Logarithm Problem (DLP)

Given:

* Prime ( p )
* Generator ( \alpha )
* Element ( \beta )

Find ( x ):


$$\alpha^x \equiv \beta \pmod{p}$$


### Example


$$5^x \equiv 41 \pmod{47}$$


Solution:


$$x = 15$$


Key property:

**DLP is computationally hard for large parameters.**

---

# Generalized Discrete Logarithm Problem

Some cyclic groups make DLP **easy**.

Example:

Group:


$$(Z_{11}, +)$$


Generator:


$$\alpha = 2$$


If we solve:


$$2x = 3 \pmod{11}$$


This is easy → therefore **not secure for cryptography**.

---

# Groups Used in Cryptography

Secure DLP groups include:

1. **Multiplicative group of prime field**


$$Z_p^*$$


2. **Elliptic Curve Groups**

3. **Galois Field Groups**


$$GF(2^m)$$


4. **Hyperelliptic curves**

---

# Attacks Against DLP

Goal:

Find ( x ) in:


$$\beta = \alpha^x$$


Two types of attacks:

## Generic Algorithms

Depend on **group size**

Examples:

* Brute-force
* Baby-step giant-step
* Pollard's rho

## Non-Generic Algorithms

Use **structure of the group**

Example:

* Index Calculus Method

---

# Brute-Force Search

Method:

Compute:


$$\alpha^1, \alpha^2, \alpha^3 ...$$


Until:


$$\alpha^x = \beta$$


Complexity:


$$O(|G|)$$


Average:


$$|G|/2$$


Security requirement:


$$|G| \ge 2^{80}$$


---

# Shanks’ Baby-Step Giant-Step Algorithm

A **time–memory tradeoff** algorithm.

Rewrite:


$$x = x_g m + x_b$$


Where:


$$0 \le x_g, x_b < m$$


### Baby Step

Compute and store:


$$\alpha^{x_b}$$


### Giant Step

Compute:


$$\beta (\alpha^{-m})^{x_g}$$


If match occurs:


$$x = x_g m + x_b$$


Complexity:


$$O(\sqrt{|G|})$$


Security recommendation:


$$|G| \approx 2^{160}$$


---

# Pollard’s Rho Method

Similar runtime to **Baby-Step Giant-Step**, but with **very small memory**.

Method:

Generate elements:


$$\alpha^i \beta^j$$


Until **collision occurs**.

Then solve for:


$$x$$


Important:

Best practical algorithm for **Elliptic Curve DLP**.

---

# Pohlig–Hellman Algorithm

Uses **Chinese Remainder Theorem (CRT)**.

If:


$$|G| = p_1^{e_1} p_2^{e_2} ... p_l^{e_l}$$


Compute:


$$x_i = x \mod p_i^{e_i}$$


Then reconstruct **x using CRT**.

Security rule:

The **smallest prime factor of |G|** must be large:


$$\ge 2^{160}$$


---

# Diffie–Hellman Key Exchange (DHKE)

Proposed by:

**Diffie & Hellman (1976)**

Solves **key distribution problem**.

Used in:

* SSH
* TLS
* IPSec

---

# Diffie–Hellman Setup

1. Choose large prime **p**
2. Choose generator **α**
3. Publish **p, α**

### Key Exchange

Alice:


$$A = \alpha^a \mod p$$


Bob:


$$B = \alpha^b \mod p$$


Shared key:


$$K = \alpha^{ab} \mod p$$


---

# Security of Diffie–Hellman

DHKE security relies on:

### Diffie–Hellman Problem (DHP)

Given:


$$A = \alpha^a, B = \alpha^b$$


Compute:


$$\alpha^{ab}$$


DHP is believed hard because of **DLP**.

Important note:

Basic DHKE is vulnerable to **Man-in-the-Middle attack (MITM)**.

---

# ElGamal Encryption

Proposed by **Taher ElGamal (1985)**.

Based on **Diffie–Hellman**.

Works over:


$$Z_p^*$$


Also works in:

* ( GF(2^m) )
* elliptic curve groups

---

# ElGamal Key Generation

Choose:

* Prime ( p )
* Generator ( \alpha )

Private key:


$$d$$


Public key:


$$\beta = \alpha^d \mod p$$


Public key = **(p, α, β)**

---

# ElGamal Encryption

Sender chooses random **i**

Compute:


$$k_E = \alpha^i \mod p$$


Masking key:


$$k_M = \beta^i \mod p$$


Ciphertext:


$$y = x \cdot k_M \mod p$$


Final ciphertext:


$$(k_E, y)$$


Important:

ElGamal is **probabilistic**.

Same plaintext → **different ciphertexts**.

---

# ElGamal Decryption

Receiver computes:


$$k_M = k_E^d \mod p$$


Recover message:


$$x = y \cdot k_M^{-1} \mod p$$


Using:

* **Extended Euclidean Algorithm**
* **Fermat's theorem**


$$k_M^{p-1} \equiv 1 \mod p$$


---

# Computational Aspects

## Key Generation

* Generate prime ( p ) (≥1024 bits)
* Compute exponentiation using **Square-and-Multiply**

## Encryption

Requires:

* 2 exponentiations
* 1 modular multiplication

## Decryption

Requires:

* 1 exponentiation
* 1 modular inverse

---

# Security of ElGamal

## Passive Attacks

Recover message from:


$$p, \alpha, \beta, k_E, y$$


Depends on difficulty of:

* **DLP**
* **DHP**

## Active Attacks

Public keys must be **authenticated**.

Solution:

* **Digital certificates**

Important rule:

**Random exponent i must never be reused**.

Otherwise:


$$k_M = \beta^i$$


would repeat → message leakage.
