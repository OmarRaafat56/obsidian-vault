Computer Architecture: Week 3 - Sequential Logic
## 1. Introduction to Sequential Logic

Unlike Combinational Logic, where the output depends solely on current inputs, Sequential Logic produces outputs based on both current and past input values.

    Memory: The ability of a circuit to store information.

    State: A "snapshot" of all relevant elements in a system at a specific moment.

## 2. Basic Storage Elements
### Cross-Coupled Inverters

The simplest storage form. It has two stable states (Q=0 or Q=1) but lacks a control mechanism to change the value easily.
R-S Latch (Reset-Set)

Constructed using cross-coupled NAND gates.

    Set (S=0): Forces Q to 1.

    Reset (R=0): Forces Q to 0.

    Quiescent State (S=1,R=1): Holds the previous value.

    Forbidden State (S=0,R=0): Breaks the Q=¬Q′ invariant and can lead to metastability (unpredictable oscillation).

### Gated D Latch (Level-Triggered)

Solves the R-S Latch forbidden state by adding logic.

    Data (D): The value to be stored.

    Write Enable (WE): When WE=1, Q follows D. When WE=0, Q is "latched" (frozen).

    Limitation: It is transparent—any change in D while WE is high immediately affects Q.

### D Flip-Flop (Edge-Triggered)

The building block of modern synchronous systems.

    Created by connecting two D Latches in a Master-Slave configuration.

    Function: Samples the input D only at the rising edge of the clock (0→1).

    Benefit: Q remains stable for the entire clock cycle, preventing race conditions.

## 3. Registers and Memory
Registers

A collection of D Flip-Flops that store multiple bits (e.g., a 4-bit register using 4 flip-flops) sharing a single clock and enable signal.
Memory Arrays

A collection of storage locations, each identified by a unique address.

    Address Space: The total number of unique locations (determined by 2n where n is the number of address bits).

    Addressability: The number of bits stored at each location.

    Components: Address Decoders (to select rows) and Multiplexers (to select outputs).

## 4. Finite State Machines (FSM)

An FSM is a discrete-time model used to design complex sequential logic.
Five Core Elements

    States: A finite number of possible conditions.

    Inputs: External signals triggering transitions.

    Outputs: Signals generated by the machine.

    State Transitions: Rules for moving between states.

    Output Specification: Rules for determining output values.

### Moore vs. Mealy Machines
Feature	Moore FSM	Mealy FSM
Output depends on...	Current State only	Current State AND Inputs
Visual Hint	Output is inside the state circle	Output is on the transition arc
